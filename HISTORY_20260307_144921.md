## 2026-03-07 对话记录（续2）

### 新增：Docker 部署方案

**用户需求**：担心影响VPS目录结构，想用Docker部署

**实现方案**：

#### 1. 创建文件

- **Dockerfile**: 基于 Python 3.12 Alpine，包含 Chromium
- **docker-compose.yaml**: 定义服务、卷映射、资源限制
- **docker-cron.sh**: 容器内定时任务脚本
- **.dockerignore**: 忽略不必要文件
- **requirements.txt**: Python依赖
- **DOCKER_DEPLOY.md**: 详细部署指南

#### 2. 数据持久化

使用 Docker 卷映射到当前目录：

| 目录 | 内容 |
|------|------|
| `./data` | 进度、配置 |
| `./html` | HTML缓存 |
| `./json` | JSON数据 |
| `./output` | 输出文件 |
| `./dongchedi` | 懂车帝数据 |

**删除容器不丢失数据**，只有删除这些目录才会。

#### 3. 定时任务

容器内使用 shell 脚本模拟 cron：
- 周一、周四 2:00：汽车之家
- 周二、周五 2:00：懂车帝
- 周三、周六 3:00：合并数据
- 随机延迟 0-60 分钟

#### 4. 使用方式

```bash
# 自动定时运行
docker-compose up -d crawl-cron

# 手动运行
docker-compose --profile manual up -d crawl-manual
docker-compose exec crawl-manual bash
python test_autohome.py --step 1 --auto
```

#### 5. 优势

| 特性 | Docker | VPS直接部署 |
|------|--------|-------------|
| 环境隔离 | ✅ 完全隔离 | ❌ 共享环境 |
| 部署难度 | ⭐ 简单 | ⭐⭐ 中等 |
| 数据管理 | 目录映射 | 直接访问 |
| 推荐场景 | **推荐** | 已有环境 |

#### 6. 资源限制

默认配置：
- CPU: 最大 2 核
- 内存: 最大 2GB

可在 docker-compose.yaml 中调整。
